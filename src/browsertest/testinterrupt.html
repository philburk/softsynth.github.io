Test interrupt() in Java Applet

This Java program demonstrates an apparent bug in the Netscape Java VM for Communicator 4.05 (and maybe other versions).

Bug: calling interrupt() on a thread appears to have NO effect under Netscape.

This program runs a Thread that sleeps for two seconds then prints "Woke Up".

When you click on the "Interrupt" button, it should interrupt() the thread causing it to print "Interrupted!".

Under NetScape Communicator 4.05-4.73, clicking on the "Interrupt" button has no effect!
The program works fine under Internet Explorer 4.0
