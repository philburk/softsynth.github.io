For  JSyn Plugins on Macintosh with Netscape Communicator

Known Bugs

The implementation of Java under Netscape V4.x on the Macintosh is considered by many developers to be badly broken. JSyn often hangs under Netscape. I have tried to debug this but can find no bugs in JSyn that could cause this hang. If Netscape hangs when using JSyn then please consider using the iCab browser instead. Hopefully the next version of Java under Netscape will work better.

Troubleshooting

If Netscape cannot find a JSyn class, then please check the following:
  1. Are the files "npjsyn" or "npjsyn.jar" installed in the"Plug-ins"  folder for Netscape? If not, please install the JSyn plugin for Macintosh.
  2. Did you restart your browser after installing those files?
  3. Did you see a new window open up that mentioned EMBEDing the JSyn plugin? If not, the web site designer may be using the old technique of EMBEDing the Jsyn plugin directly in the web page.
If you still have problems make sure you have the most recent versions of JSyn and Netscape..

Turning on the Java Console

You can get more information and error messages from the browser if you enable the Java Console. Please cut and paste these error messages when reporting bugs. Select the "Tools/Java Console" Item from the "Communicator" menu.

If you cannot find an answer to your question on this website, then please contact us directly.
