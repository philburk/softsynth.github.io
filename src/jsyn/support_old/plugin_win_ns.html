For  JSyn Plugins on Windows (PCs) with Netscape Communicator

Known Bugs

The Netscape browser can periodically unload and then reload a plugin when switching windows. If JSyn tries to call the plugin at that moment, then Netscape can hang. I have provided a JavaScript solution that seems to prevent the problem if used in a website containing JSyn Applets.

The JSyn plugin does not appear to work under the new Netscape 6 because they changed their interface! The JSyn plugin will be updated ASAP to conform to the new Netscape. In the meantime use Netscape 4.x.

Troubleshooting

If Netscape cannot run a JSyn Applet, then please check the following:
  1. Are the files "npjsyn.dll" and "npjsyn.jar" installed in the"Netscape\Communicator\Program\Plugins" folder? If not, please install the JSyn plugin for Netscape on PC.
  2. Did you restart your browser after installing those files?
  3. Did you see a new window open up that mentioned EMBEDing the JSyn plugin? If not, the web site designer may be using the old technique of EMBEDing the Jsyn plugin directly in the web page.
If you still have problems I suggest using the most recent versions of JSyn and version 4.73 of Netscape..

Turning on the Java Console

You can get more information and error messages from the browser if you enable the Java Console. Please cut and paste these error messages when reporting bugs. Select the "Tools/Java Console" Item from the "Communicator" menu.

If you cannot find an answer to your question on this website, then please contact us directly.
