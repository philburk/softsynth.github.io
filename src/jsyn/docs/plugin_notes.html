Notes on How InternetExplorer and Netscape Use Plugins

Through experimentation, I have made the following observations relating to IE V5 and Netscape V4.61 and the use of Applets that use native code.

Windows

  1. Applets with native code can be made to run on IE under Windows by putting the classes  in a JAR file in the CLASSPATH, and by putting the native JNI DLL in the PATH.
  2. Applets running in IE will call classes in a JAR file referenced in the CLASSPATH. But when Netscape is run, it also picks up the JAR file from the CLASSPATH instead of the JAR file in the Plugins directory. This can cause problems making it difficult to configure a system to use JSyn in both IE and Netscape under Windows.
  3.  IE on Windows will load a Netscape Plugin if EMBEDed in a web page but it does not appear to call it using LiveConnect. When exiting the page a "plugin error" is reported in an alert. Because of this it is recommended to only EMBED the plugin when using Netscape by using JavaScript to detect the browser type.

Macintosh

  1. Applets with native code can be made to run on IE and iCab on the Macintosh by putting the classes containing JDirect calls in a JAR file in the "System Folder:Extensions:MRJ Libraries:MRJCLasses" folder, and by putting the native code in the "System Folder:Extensions" folder.
  2.  IE on Macintosh will fail to load the Netscape Plugin if EMBEDed in a web page and report an error in an alert. Because of this it is recommended to only EMBED the plugin when using Netscape by using JavaScript to detect the browser type.
  3. IE on the Macintosh will crash if it sees the JSyn plugins in the Netscape:Plug-ins folder! This can cause problems making it difficult to configure a system to use JSyn in both IE and Netscape on Macintosh.
