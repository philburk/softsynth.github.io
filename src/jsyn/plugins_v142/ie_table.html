Hypothesis

I am starting to think there are several reasons JSyn won't work under IE.
  1. JSyn fails on NT because of DirectSound inadequacies. (Workaround found for NT bug. Get JSyn V13.5 if using NT.)
  2. JSyn does not work under IE V4.0 unless one installs the new V5.0 Java Virtual Machine.
  3. JSyn V13.4 2/16 does not work on some peoples machines because I made the JAR file using JDK 1.3 which does not produce a manifest.
  4. JSyn fails if the Netscape plugin is installed, IE tries to load it and gets confused because it does not support LiveConnect.
Please let me know if you see a situation that contradicts this hypothesis.
